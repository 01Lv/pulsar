Set up a standalone Pulsar locally

对于本地开发和测试，你可以在计算机上独立运行 Pulsar。 独立模式包含 Pulsar broker、必须的 Zookeeper 和 BookKeeper 组件，它们运行在一个单独的 Java 虚拟机（JVM）进程之中。

Pulsar运行在生产环境?

如果你想在生产环境运行完整的Pulsar，请参考 Pulsar的集群部署

安装 Pulsar 独立模式

本教程指导将为你讲解安装过程的每一步。

系统要求

Pulsar 目前可用于** MacOS 和 Linux **。 要使用Pulsar，您需要从Oracle下载中心安装 Java 8。

Tip

默认情况下，Pulsar 分配 2G JVM 堆内存来启动。 可以在PULSAR_MEM下的conf / pulsar_env.sh文件中进行更改。 这是传递给 JVM 的额外选项。

使用二进制版本安装 Pulsar

要开始使用 Pulsar，请通过以下方式之一下载二进制 tarball 版本：

  • 从 Apache 镜像下载(Pulsar 2.4.1 二进制版本)

  • 从 Pulsar 官网下载页

  • 从 Pulsar 下载发布页面

  • 使用wget：

    $ wget https://archive.apache.org/dist/pulsar/pulsar-2.4.1/apache-pulsar-2.4.1-bin.tar.gz

下载压缩包后，解压缩并使用cd命令导航到结果目录：

$ tar xvfz apache-pulsar-2.4.1-bin.tar.gz
$ cd apache-pulsar-2.4.1

软件包包含的内容:

Pulsar二进制包包含下列目录:

目录包含
binPulsar 的命令行工具，比如 pulsar 和 pulsar-admin.
confPulsar 的配置文件，包含 broker 配置,ZooKeeper 配置 等等.
examples一个Java JAR 包，包含 Pulsar Functions 的例子.
libPulsar使用到的 JAR 文件
licenses开源许可文件 .txt格式, 适用于Pulsar 代码库 的各种组件.

这些目录在你运行 Pulsar 的时候生成.

目录包含
dataZooKeeper和BookKeeper使用的数据存储目录
instances为 Pulsar Functions 创建的组件.
logs安装时生成的日志文件

Tip

If you want to use builtin connectors and tiered storage offloaders, you can install them according to the following instructions：

  • 安装内置连接器（可选）
  • 安装分层存储卸载程序（可选）

Otherwise, skip this step and perform the next step Start Pulsar standalone. Pulsar can be successfully installed without installing bulitin connectors and tiered storage offloaders.

安装内置连接器（可选）

自从 2.1.0-孵化 发布以来，Pulsar发布了一个单独的二进制分布，其中包含所有 内置的 连接器。 要启用这些 内置的 连接器，可以以下列方式下载连接器 tarball 释放：

  • 从Apache镜像下载 Pulsar IO 连接器{{pulsar：version}}版本

  • 从 Pulsar 官网下载页

  • 从 Pulsar 下载发布页面

  • 使用wget：

    $ wget https://archive.apache.org/dist/pulsar/pulsar-2.4.1/connectors/{connector}-2.4.1.nar

下载完 nar 文件后，将文件复制到 pulsar 目录中的 连接器 目录。 例如，如果你下载 pulsar-io-aerospike-2.4.1.nar 连接器文件，请输入以下命令：

$ mkdir connectors
$ mv pulsar-io-aerospike-2.4.1.nar connectors

$ ls connectors
pulsar-io-aerospike-2.4.1.nar
...

Note

  • If you are running Pulsar in a bare metal cluster, make sure connectors tarball is unzipped in every pulsar directory of the broker (or in every pulsar directory of function-worker if you are running a separate worker cluster for Pulsar Functions).

  • If you are running Pulsar in Docker or deploying Pulsar using a docker image (e.g. K8S or DCOS), you can use the apachepulsar/pulsar-all image instead of the apachepulsar/pulsar image. apachepulsar/pulsar-all 镜像已经捆绑了 所有内置连接器。

安装分层存储卸载程序（可选）

Tip

Since 2.2.0 release, Pulsar releases a separate binary distribution, containing the tiered storage offloaders. To enable tiered storage feature, follow the instructions below; otherwise skip this section.

要开始使用 分层存储卸载程序，你需要通过以下一种方式在每个代理节点上下载卸载程序的 tarball 版本：

  • 从 Apache镜像下载 Pulsar IO 连接器{{pulsar：version}}版本

  • 从 Pulsar 官网下载页

  • 从 Pulsar 下载发布页面

  • 使用wget：

    $ wget https://archive.apache.org/dist/pulsar/pulsar-2.4.1/apache-pulsar-offloaders-2.4.1-bin.tar.gz

下载完压缩包后，解压缩卸载程序包，并将卸载程序复制为offloaders 在 pulsar 目录中：

$ tar xvfz apache-pulsar-offloaders-2.4.1-bin.tar.gz

// you will find a directory named `apache-pulsar-offloaders-2.4.1` in the pulsar directory
// then copy the offloaders

$ mv apache-pulsar-offloaders-2.4.1/offloaders offloaders

$ ls offloaders
tiered-storage-jcloud-2.4.1.nar

有关如何配置分层存储的更多信息，请参见 分层存储指南。

Note

  • If you are running Pulsar in a bare metal cluster, make sure that offloaders tarball is unzipped in every broker's pulsar directory.

  • If you are running Pulsar in Docker or deploying Pulsar using a docker image (e.g. K8S or DCOS), you can use the apachepulsar/pulsar-all image instead of the apachepulsar/pulsar image. apachepulsar/pulsar-all image has already bundled tiered storage offloaders.

安装 Pulsar 独立模式

拥有该发行版的最新本地副本后，可以使用 pulsar</a启动本地集群 >命令，该命令存储在bin目录中，并指定你要以独立模式启动Pulsar。

$ bin/pulsar standalone

如果成功启动了 Pulsar，你将看到如下所示的INFO级日志消息：

2017-06-01 14:46:29,192 - INFO  - [main:WebSocketService@95] - Configuration Store cache started
2017-06-01 14:46:29,192 - INFO  - [main:AuthenticationService@61] - Authentication is disabled
2017-06-01 14:46:29,192 - INFO  - [main:WebSocketService@108] - Pulsar WebSocket Service started

Tip

  • The service is running on your terminal, which is under your direct control. If you need to run other commands, open a new terminal window.
    You can also run the service as a background process using the pulsar-daemon start standalone command. For more information, see pulsar-daemon.

  • By default, there is no encryption, authentication, or authorization configured. Apache Pulsar 可以在没有任何授权的情况下从远程服务器访问。 请检查 安全概述 文档以确保你的部署。

  • When you start a local standalone cluster, a public/default namespace is created automatically. 命名用于开发。 所有Pulsar的topic主题都在命名空间中进行管理。 更多信息请参考 Topics.

独立使用 Pulsar

Pulsar 提供一个名为 pulsar-client 的 CLI 工具。 Pulsar-client 工具使你能够在运行的集群中消费并生成到 Pulsar 主题的消息。

消费一条消息

以下命令会消耗一个消息，订阅名称为 首次订阅 到 我的主题 主题：

$ bin/pulsar-client consume my-topic -s "first-subscription"

如果消息成功发送给了 topic，你应该会在 pulsar-client 日志中找到一个确认， 类似：

09:56:55.566 [pulsar-client-io-1-1] INFO  org.apache.pulsar.client.impl.MultiTopicsConsumerImpl - [TopicsConsumerFakeTopicNamee2df9] [first-subscription] Success subscribe new topic my-topic in topics consumer, partitions: 4, allTopicPartitionsNumber: 4

Tip

As you have noticed that we do not explicitly create the my-topic topic, to which we consume the message. When you consume a message to a topic that does not yet exist, Pulsar creates that topic for you automatically. Producing a message to a topic that does not exist will automatically create that topic for you as well.

产生一条消息

这条命令将发送一条简单的消息 hello-pulsar 给 my-topic topic:

$ bin/pulsar-client produce my-topic --messages "hello-pulsar"

如果消息成功发送给了 topic，你应该会在 pulsar-client 日志中找到一个确认， 类似：

13:09:39.356 [main] INFO  org.apache.pulsar.client.cli.PulsarClientTool - 1 messages successfully produced

停止 Pulsar 独立模式

按 Ctrl+C 停止本地独立的 Pulsar。

Tip

If the service runs as a background process using the pulsar-daemon start standalone command, then use the pulsar-daemon stop standalone command to stop the service.

For more information, see pulsar-daemon.
