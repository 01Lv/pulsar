Pulsar SQL 配置和部署

你可以配置 Presto Pulsar 連接器，並通過以下說明部署一個集群。

設定 Presto Pulsar Connector

你可以在 ${project.root}/conf/presto/catalog/pulsar.properties 屬性文件中配置 Presto Pulsar 連接器。 連接器的配置及其預設值如下所述。

# 要在 catalog 中顯示的連接器的名稱
connector.name=pulsar

# Pulsar broker 服務 URL
pulsar.broker-service-url=http://localhost:8080

# Zookeeper 集群 URI
pulsar.zookeeper-uri=localhost:2181

# 單次讀取的最少信息數
pulsar.entry-read-batch-size=100

# 每個查詢使用的預設拆分數
pulsar.target-num-splits=4

你可以使用 Presto 連接到有多個主機的 Pulsar 集群。 要為 broker 配置多個主機, 添加多個 URL 到 pulsar.broker-service-url。 要為 ZooKeeper 配置多個主機, 添加多個 URI 到 pulsar.zookeeper-uri。 以下是一個範例。

pulsar.broker-service-url=http://localhost:8080,localhost:8081,localhost:8082
pulsar.zookeeper-uri=localhost1,localhost2:2181

在現有的 Presto 集群進行 Pulsar 查詢

如果已經有了一個 Presto 集群, 你可以將 Presto Pulsar 連接器插件複製到現有的群集上。你可以通過以下方式下載插件 套件包。

$ wget https://archive.apache.org/dist/pulsar/pulsar-2.6.0/apache-pulsar-2.6.0-bin.tar.gz

部署新群集

Since Pulsar SQL is powered by Presto, the configuration for deployment is the same for the Pulsar SQL worker.

注意
關於如何設置一個獨立的單個節點環境，請參閱 查詢數據。

你可以使用相同的 CLI 參數給 Presto 啟動器：

$ ./bin/pulsar sql-worker --help
Usage: launcher [options] command

Commands: run, start, stop, restart, kill, status

Options:
  -h, --help            show this help message and exit
  -v, --verbose         Run verbosely
  --etc-dir=DIR         Defaults to INSTALL_PATH/etc
  --launcher-config=FILE
                        Defaults to INSTALL_PATH/bin/launcher.properties
  --node-config=FILE    Defaults to ETC_DIR/node.properties
  --jvm-config=FILE     Defaults to ETC_DIR/jvm.config
  --config=FILE         Defaults to ETC_DIR/config.properties
  --log-levels-file=FILE
                        Defaults to ETC_DIR/log.properties
  --data-dir=DIR        Defaults to INSTALL_PATH
  --pid-file=FILE       Defaults to DATA_DIR/var/run/launcher.pid
  --launcher-log-file=FILE
                        Defaults to DATA_DIR/var/log/launcher.log (only in
                        daemon mode)
  --server-log-file=FILE
                        Defaults to DATA_DIR/var/log/server.log (only in
                        daemon mode)
  -D NAME=VALUE         Set a Java system property

集群的預設設定檔位於 ${project.root}/conf/presto。你可以通過修改預設的配置來客制你的部署。

你還可以讓 worker 從指定的目錄中來讀取設定參數檔，或設定寫出數據的目錄：

$ ./bin/pulsar sql-worker run --etc-dir /tmp/incubator-pulsar/conf/presto --data-dir /tmp/presto-1

你可以將 worker 啟動為 daemon 程序。

$ ./bin sql-worker start

在多個節點上部署 Pulsar SQL 集群

你可以在多個節點上部署 Pulsar SQL 集群或 Presto集群。以下範例展示如何在三個節點上部署集群。

  1. 將 Pulsar binary 發行版複製到三個節點上。

第一個節點作為 Presto coordinator。 最少的配置要求如下 (設定檔: ${project.root}/conf/presto/config.properties )。

coordinator=true
node-scheduler.include-coordinator=true
http-server.http.port=8080
query.max-memory=50GB
query.max-memory-per-node=1GB
discovery-server.enabled=true
discovery.uri=<coordinator-url>

其他兩個節點用來作為 worker 節點，可以對 worker 節點使用以下配置。

coordinator=false
http-server.http.port=8080
query.max-memory=50GB
query.max-memory-per-node=1GB
discovery.uri=<coordinator-url>

  1. 修改每一個節點 ${project.root}/conf/presto/catalog/pulsar.properties 設定檔裡頭的 pulsar.broker-service-url 與 pulsar.zookeeper-uri。

  2. 啟動 coordinator 節點。

    $ ./bin/pulsar sql-worker run

  1. 啟動 worker 節點。

    $ ./bin/pulsar sql-worker run

  1. 啟動 SQL CLI 並檢查集群的狀態。
$ ./bin/pulsar sql --server <coordinate_url>
  1. 檢查節點的狀態。
presto> SELECT * FROM system.runtime.nodes;
 node_id |        http_uri         | node_version | coordinator | state  
---------+-------------------------+--------------+-------------+--------
 1       | http://192.168.2.1:8081 | testversion  | true        | active 
 3       | http://192.168.2.2:8081 | testversion  | false       | active 
 2       | http://192.168.2.3:8081 | testversion  | false       | active

For more information about deployment in Presto, refer to Presto deployment.

Note
The broker does not advance LAC, so when Pulsar SQL bypass broker to query data, it can only read entries up to the LAC that all the bookies learned. You can enable periodically write LAC on the broker by setting "bookkeeperExplicitLacIntervalInMills" in the broker.conf.
