Schema 自動更新

如果某個 schema 通過了 schema 相容性檢查，在預設的情形下 producer 會自動將此 schema 更新到要發佈信息的 topic 上。

Producer 端的自動更新

對 producer 來說 AutoUpdate 發生在以下情況：

  • 如果 topic 沒有 schema，則 Pulsar 會自動註冊一個 schema。

  • 如果 topic 有 schema：

    • 如果 producer 沒有隨附 schema：

      • 如果在 topic 所屬的命名空間中不啟用 isSchemaValidationEnforced 或 schemaValidationEnforced ，則允許 producer 連接到該 topic 並產生數據。

      • 如果在 topic 所屬的命名空間中** 啟用了**isSchemaValidationEnforced或schemaValidationEnforced 則 producer 將被拒絕並斷開連接。

    • 如果 producer 隨付 schema：

      Broker 根據 topic 所屬的命名空間的**相容性檢查策略**設定來執行相容性檢查。

      • 如果 schema 已被註冊，則 producer 將連接到 broker。

      • 如果 schema 未註冊：

        * 如果 `isAllowAutoUpdateSchema` 設定為 ** false **，則 producer 將被拒絕連接到 broker。

* 如果 `isAllowAutoUpdateSchema` 設定為 **true**：
    
    * 如果 schema 通過了相容性檢查，那麼 broker 將為該 topic 自動註冊一個新的 schema，並且允許 producer 來進行連接。
    
    * 如果 schema 沒通過相容性檢查，那麼 broker 不會註冊新的 schema, 並且不允許 producer 來進行連接。

Consumer 端的自動更新

對 consumer 來說 AutoUpdate 會發生以下情況：

  • 如果 consumer 在不隨附 schema 的情況下連接到 topic（這意味著 consumer 可接收 raw bytes），則 consumer 可以成功連接到該 topic 而無需進行任何相容性檢查。

  • 如果** consumer 隨附 schema 連接到 topic**：

    • 如果 topic 沒有全部條件 ( schema /數據/本地 consumer 和本地 producer ）：

      • 如果 isAllowAutoUpdateSchema 設定為 ** true**，那麼 consumer 將註冊這個 schema，並將連接到broker。

      • 如果 isAllowAutoUpdateSchema 設定為** false **，則 consumer 將被拒絕連接到 broker。

    • 如果 topic 有任一條件符合 ( schema /數據/本地 consumer 和本地 producer ），則將執行 schema 相容性檢查。

      • 如果 schema 通過相容性檢查，那麼 consumer 就將連接到 broker。

      • 如果該 schema 沒通過相容性檢查，則 consumer 將被拒絕連接到 broker。

管理自動更新策略

你可以使用 pulsar-admin 命令來管理 AutoUpdate策略，如下所示：

  • 啟用 AutoUpdate

  • 關閉 AutoUpdate

  • 調整兼容性

啟用 AutoUpdate

要在命名空間上啟用 AutoUpdate，可以使用 pulsar-admin 命令工具。

bin/pulsar-admin namespaces set-is-allow-auto-update-schema --enable tenant/namespace

關閉 AutoUpdate

要在命名空間上關閉 AutoUpdate，可以使用 pulsar-admin 命令工具。

bin/pulsar-admin namespaces set-is-allow-auto-update-schema --disable tenant/namespace

關閉 AutoUpdate 後，你只能使用 pulsar-admin 命令工具來手動註冊新的 schema。

調整相容性

要調整命名空間上的 schema 相容性檢查級別，可以使用 pulsar-admin命令工具。

bin/pulsar-admin namespaces set-schema-compatibility-strategy --compatibility <compatibility-level> tenant/namespace

Schema 驗證

預設情況下，producer 的 schemaValidationEnforced 的功能是被關閉的：

  • 這意味著沒有隨附 schema 的 producer 可以對帶有 schema 的 topic 發送任何類型的信息，這可能會導致該 topic 裡累積很多垃圾數據。

  • 當然這樣設定也就可以讓那些不支持 schema 機制的非 Java 語言客戶端可以對帶有 schema 的 topic 發佈信息。

但是，如果你希望對帶有 schema 的 topic 有更強的資料相容保證，則可以在整個群集中或在每個命名空間的基礎上啟用 schemaValidationEnforced。

啟用 schema 驗證

要在命名空間上啟用 schemaValidationEnforced，可以使用 pulsar-admin 命令工具。

bin/pulsar-admin namespaces set-schema-validation-enforce --enable tenant/namespace

關閉 Schema 驗證

要在命名空間上關閉 schemaValidationEnforced，可以使用 pulsar-admin 命令工具。

bin/pulsar-admin namespaces set-schema-validation-enforce --disable tenant/namespace

上傳 schema

要對某一個 topic 上傳 (註冊) 一個新的 schema，你可以使用以下方法之一。

Admin CLI
REST API
Java Admin API

$ pulsar-admin schemas upload --filename <schema-definition-file> <topic-name>

這個 schema-definition-file 是使用 JSON 格式來定義。

{
    "type": "<schema-type>",
    "schema": "<an-utf8-encoded-string-of-schema-definition-data>",
    "properties": {} // the properties associated with the schema
}

擷取 schema

要獲取 topic 的 schema，可以使用以下方法之一。

Admin CLI

使用 pulsar-admin 的 extract 子命令。

$ pulsar-admin schemas extract --classname <class-name> --jar <jar-path> --type <type-name>

刪除 schema

要刪除 topic 的 schema，可以使用以下方法之一。

Schema 存儲客制化

預設情況下，Pulsar 在 Apache BookKeeper 中保存各種類型的數據 schema 和 Pulsar 設定。

然而，你可以在需要時使用另一個存儲系統。

實現

若要使用 Pulsar schemas 的非預設 (非 BookKeeper) 存儲系統，需要實現下列 Java 介面接口：

  • SchemaStorage 介面

  • SchemaStorageFactory 介面

SchemaStorage 介面

` SchemaStorage </ code>介面接口具有以下 method：

public interface SchemaStorage {
    // How schemas are updated
    CompletableFuture put(String key, byte[] value, byte[] hash);

// How schemas are fetched from storage
CompletableFuture<StoredSchema> get(String key, SchemaVersion version);

// How schemas are deleted
CompletableFuture<SchemaVersion> delete(String key);

// Utility method for converting a schema version byte array to a SchemaVersion object
SchemaVersion versionFromBytes(byte[] version);

// Startup behavior for the schema storage client
void start() throws Exception;

// Shutdown behavior for the schema storage client
void close() throws Exception;



}
`

提示

有關 schema 存儲實現的完整範例，請參見 BookKeeperSchemaStorage 類別。

SchemaStorageFactory 介面

SchemaStorageFactory 介面接口具有以下 method：

public interface SchemaStorageFactory {
    @NotNull
    SchemaStorage create(PulsarService pulsar) throws Exception;
}

提示

有關 schema storage factory 實現的完整範例，請參見 BookKeeperSchemaStorageFactory 類別。

部署

要使用自定義 schema 存儲元件，請依循以下步驟。

  1. 將實作打包成 JAR 檔案裡。

  2. 將 JAR 檔案添加到 Pulsar 的 lib 文件夾中。

  3. 將 broker.conf 中 schemaRegistryStorageClassName 的設定更改為自定義 factory 類別。

  4. 啟動 Pulsar。
