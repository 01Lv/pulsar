Managing Tenants

テナントはネームスペースのようなもので、管理 API を使って管理することができます。いまのところテナントについて二つの設定が行えます。

  • Admin roles
  • Allowed clusters

テナントのリソース

一覧

pulsar-admin

list サブコマンドを使用するとインスタンスに関連づけられたすべてのテナントの一覧を表示できます。

$ pulsar-admin tenants list

これは次のような単純なリストを返します。

my-tenant-1
my-tenant-2

作成

pulsar-admin

新しいテナントは create サブコマンドで作成できます。

$ pulsar-admin tenants create my-tenant

テナントを作成する際に、-r/--admin-roles フラグで管理者ロールの割り当てが行えます。カンマ区切りにより複数のロールを指定できます。設定例は以下のとおりです。

$ pulsar-admin tenants create my-tenant \
  --admin-roles role1,role2,role3

$ pulsar-admin tenants create my-tenant \
  -r role1

設定の取得

pulsar-admin

get サブコマンドにテナント名を渡すとそのテナントの設定をJSON形式で見ることができます。

$ pulsar-admin tenants get my-tenant
{
  "adminRoles": [
    "admin1",
    "admin2"
  ],
  "allowedClusters": [
    "cl1",
    "cl2"
  ]
}

削除

pulsar-admin

delete サブコマンドにテナント名を渡すとそのテナントを削除することができます。

$ pulsar-admin tenants delete my-tenant

更新

pulsar-admin

update サブコマンドを使用するとテナントの設定を更新することができます。
